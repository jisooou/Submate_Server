# Subscription Management System (Submate)

## 📖 프로젝트 소개

사용자가 다양한 구독 서비스를 한눈에 관리하고, 소비 패턴을 기반으로 맞춤형 알림을 제공하는 웹 애플리케이션이다.

구독 서비스의 증가로 인해 결제일을 잊어버리거나, 월 지출을 정확히 파악하지 못하고, 사용하지 않는 구독이 계속 결제되는 문제를 해결하기 위해 개발하였다.

---

## 🚨 문제 상황 

현대 사용자들은 여러 개의 구독 서비스를 동시에 이용하지만 다음과 같은 문제를 겪는다.

- 월간 지출 금액을 정확히 파악하기 어렵다
- 결제일을 잊어 원치 않는 자동 결제가 발생한다
- 구독 관리 자체가 번거롭다

즉, 구독은 증가했지만 이를 효율적으로 관리할 수 있는 시스템은 부족한 상태이다.

---

## 💡 해결 방법 

본 프로젝트는 다음 세 가지 방향으로 문제를 해결한다.

### 1. 구독 서비스 통합 관리

여러 구독 서비스를 하나의 시스템에서 관리할 수 있도록 설계하였다.
카테고리 기반으로 서비스를 분류하여 사용자가 직관적으로 관리할 수 있도록 하였다.

---

### 2. 자동 비용 계산

사용자의 구독 정보를 기반으로 월간 및 연간 지출을 자동으로 계산한다.
카테고리별 비용 분석을 통해 소비 패턴을 파악할 수 있도록 한다.

---

### 3. 사용자 맞춤형 알림 시스템

다음 조건을 기반으로 알림을 생성한다.

- 결제일 3일 전 (D-3)
- 6개월 이상 장기 구독
- 월 지출 3만원 초과

단순 알림이 아니라, 사용자의 소비 행동을 개선하도록 유도하는 것이 핵심이다.

---

## ⚙️ 핵심 기능 

### - 구독 서비스 등록 및 관리

- 기존 서비스 검색 후 등록 가능
- 신규 서비스 직접 추가 가능
- 서비스명 기반 카테고리 자동 분류

---

### - 소비 분석

- 전체 구독 비용 집계
- 카테고리별 비용 분리
- 월간 / 연간 비용 자동 계산
- 환율 적용을 통한 외화 구독 처리

---

### - 결제일 관리 (D-Day)

- 결제일까지 남은 날짜 자동 계산
- 사용자에게 직관적인 형태로 제공

---

### - 알림 시스템

- 소비 패턴 기반 알림 생성
- 조건 만족 시 자동 알림 저장 및 제공

---

## 🛠️ 핵심 설계 

### ❓ 문제: 실시간 알림 방식의 한계

초기에는 Firebase 기반 푸시 알림을 고려하였다.
하지만 다음과 같은 문제가 있었다.

- 로그인 기능이 없어 토큰 관리가 어려움
- 불필요한 알림이 증가할 가능성
- 구현 복잡도 증가

---

### ✋🏻 해결: Scheduler 기반 알림 시스템

매일 특정 시간에 스케줄러를 실행하여 조건을 만족하는 경우에만 알림을 생성하도록 설계하였다.

```
[Scheduler 실행]
    ↓
[구독 데이터 조회]
    ↓
[조건 검사]
    - 결제일 3일 전
    - 6개월 이상 구독
    - 월 지출 초과
    ↓
[알림 생성 및 DB 저장]
    ↓
[프론트에서 조회 후 표시]
```

이 방식은 구현이 단순하면서도 안정적인 알림 시스템을 구성할 수 있다.

---

### ❓ 문제: 중복 알림 발생

스케줄러가 주기적으로 실행되기 때문에 동일한 알림이 여러 번 생성되는 문제가 발생하였다.

---

### ✋🏻 해결: 중복 방지 로직

알림 생성 전에 동일한 알림이 이미 존재하는지 확인하는 로직을 추가하였다.

```
IF (오늘 동일 알림 존재)
    → 생성하지 않음
ELSE
    → 알림 생성
```

이를 통해 하루 1회만 알림이 생성되도록 보장하였다.
